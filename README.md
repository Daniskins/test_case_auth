test_case_auth

Учебный проект, демонстрирующий базовые механизмы аутентификации и хранения пользователей (без использования БД и JWT)  
Данные сохраняются в JSON-файлах, а управление сессиями и безопасностью реализовано средствами стандартной библиотеки Python.

-- Функционал --
- core/operations.py - основная логика программы
- core/security.py - функции безопасности (хэширование SHA3-256, генерация токенов, UUID)
- crud/ - доступ к JSON-хранилищам пользователей и сессий
- database/ - JSON-хранилища пользователей и сессий, выступающие в роли простейшей БД

-- Технологии --
Использовал только Python 3.13 и его встроенные библиотеки

-- Безопасность --
Использовал криптографию SHA3-256 для хэширования паролей и токенов, hmac.compare_digest для проверки хэшей,
secrets.token_urlsafe для генерации токенов и uuid.uuid1 для пользовательских идентификаторов

-- Запуск --
Демонстрационный сценарий запускается командой python main.py, 
а функции demo_register_users и demo_login_and_session иллюстрируют полный цикл работы backend-приложения
